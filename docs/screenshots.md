# Screenshots

Generated by `npm run screenshotter` (`scripts/screenshots.mjs`). Every
surface is captured in **light and dark** at retina (1440×900 @2x) — no
backend, database, or wasm build required.

Two surfaces are captured:

1. **The static site** — the marketing landing page (`site/index.html`) and
   docs viewer (`site/docs.html`), served from a tiny built-in Node http
   server.
2. **The app** (`src/`) — booted on a throwaway `vite` dev server with
   `VITE_USE_MOCKS=true`, so every hook serves deterministic mock data and
   nothing reaches out to Postgres, Redis or a running node.

To regenerate: `npm run screenshotter` (alias: `npm run screenshots`).

> **How the app boots with no backend.** Every data hook in `src/hooks/`
> checks `import.meta.env.VITE_USE_MOCKS === 'true'` and serves a fixed mock
> payload when set, so the whole app renders offline and deterministically.
> The screenshotter also seeds `localStorage` before first paint (theme, a
> signed-in user, and the dismissal flags for the announcement banner, cookie
> banner and onboarding tour) so one-time overlays never cover a capture.
>
> One gotcha worth keeping: `vite` is started with an explicit
> `--host 127.0.0.1`. Without it vite binds `::1`/localhost only, the
> readiness probe never succeeds, and the app capture silently skips — which
> is exactly why app routes were skipped in the previous wave.
