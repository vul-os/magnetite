Magnetite

The decentralized, self-hostable Rust game platform. Bring any server — it scales to your hardware. No cloud required.

Magnetite (Fe₃O₄) — magnetic, grounded, the foundation games are built on. A game is a content-addressed portable object. A node is generic compute that fills its own hardware. The chain is the wallet. Discovery is a phonebook, not an authority. Everything social — chat, voice, video, streaming — is a pluggable integration, not something we build. Anyone runs the single magnetite node binary; identity is a keypair; payments are non-custodial crypto.

Grounded in six differentiators

One Rust game, one node binary, decentralized by construction — not bolted on.

Capacity-elastic hosting

A node measures its own hardware — cores, RAM, bandwidth — and advertises what it can hold. Player cap is emergent from the box, never a config constant. More cores, more shards; bring more boxes and the cluster grows into a federated mesh with other operators' nodes.

Content-addressed games

A game's id is the hash of its wasm module and manifest — BlobStore serves it by hash, with no central registry row required to identify it. Iroh/BitTorrent-style distribution is the natural next step; today it's already sha256-addressable.

Deterministic WASM sandbox

Untrusted game logic compiles to wasm32-wasip1 and runs under Wasmtime with a fuel budget, a memory cap, and an epoch-interrupt wall clock. No OS randomness, no wall clock inside the guest — same (state, ordered commands, tick, seed) always produces the same result, on any host.

Replay-verified anti-cheat

Clients send inputs, never state; the server validates and steps authoritatively. Every tick's inputs and state hash land in a ReplayLog — anyone can re-simulate from scratch with verify_replay and prove tampering, no trust required.

Non-custodial crypto payments

Wallet-to-wallet checkout with atomic developer/operator splits, signed-receipt entitlements, payment channels for per-seat hosting fees, and optional wager escrow settled by replay verification. No balances, no payouts, no custody — the chain is the wallet.

Pluggable decentralized comms

Chat, voice, video, and streaming are adapters, not a product we build: Matrix/Element for text and presence, Jitsi or LiveKit for voice/video, Owncast/PeerTube for live and VOD. The node mints scoped credentials from your keypair — one login, SSO into every room.

The seams

Everything provider-specific plugs in behind six traits. The game runtime, scheduler, and payment path never name a provider type — only the seam.

Identity

Keypair auth

Ed25519 challenge/response by default; optional DMTAP-Auth for decentralized login and key rotation behind a feature flag.

Naming

Phonebook, not authority

Raw pubkeys and short-hash addresses by default; human name@domain is a display layer over the same raw key.

BlobStore

Content-addressed assets

Hash-in, hash-out. Local + HTTP by default; Iroh/BitTorrent and MOTE/DMTAP-PUB adapters slot in without touching the game runtime.

Discovery

The phonebook

A dumb, swappable HTTP tracker anyone can run, plus LAN mDNS — never a central server-registry authority.

CommsProvider

Chat, voice, video, streaming

Matrix, Jitsi, LiveKit, Owncast/PeerTube — the node mints scoped join credentials from your keypair, optionally gated behind a payment receipt.

PaymentRail

Non-custodial crypto

Atomic checkout splits, per-seat hosting channels, wager escrow settled by replay verification. A deterministic mock rail keeps CI fully offline.

Quick start

One binary. Bring a server — it scales to whatever hardware you give it.

# install once
cargo install magnetite-cli

# scaffold a crate implementing AuthoritativeGame
magnetite new my-game
cd my-game

# cargo build --release --target wasm32-wasip1 → game.wasm
magnetite build

# build → WasmExecutor → server, ZERO backend
magnetite dev
# ws://127.0.0.1:<port> — play it right now

# bring your own box: measures its hardware,
# advertises capacity, joins the mesh
magnetite serve --advertise tracker.example.org

  3. 3
    Bring any serverPoint magnetite serve at hardware you own — a laptop, a rented box, a rack — and it measures capacity and joins the discovery mesh. No cloud, no provisioning API.
