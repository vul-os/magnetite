# Screenshots

Generated by `npm run screenshotter` (`scripts/screenshots.mjs`). Every
surface is captured in **light and dark** at retina (1440×900 @2x) from the
static marketing site and docs viewer — no backend, database, or wasm build
required.

| File | Surface |
|------|---------|
| screenshots/landing-light.png / -dark.png | Landing page — hero, features, seams, quick start |
| screenshots/docs-overview-light.png / -dark.png | Docs — Overview |
| screenshots/architecture-light.png / -dark.png | Docs — Architecture (node / discovery / payment / comms diagram) |
| screenshots/hosting-a-server-light.png / -dark.png | Docs — Hosting a server |
| screenshots/payments-light.png / -dark.png | Docs — Payments |
| screenshots/comms-light.png / -dark.png | Docs — Comms |

To regenerate: `npm run screenshotter` (alias: `npm run screenshots`).

> **Note on scope.** This wave captures the static marketing site and docs
> viewer only — they need nothing but a browser and a local static file
> server. The React app in `src/` is the pre-decentralization marketplace
> frontend (fiat wallet, communities, marketplace) and either needs a live
> Postgres/Redis backend or does not yet reflect the decentralized vision
> described in [`DECENTRALIZATION.md`](../DECENTRALIZATION.md). Capturing its
> routes is deferred to a later wave once the frontend is rebuilt against the
> new seams — see the `TODO` at the bottom of `scripts/screenshots.mjs`.
